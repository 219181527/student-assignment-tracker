# 📊 GitHub Project Template Analysis

| Template          | Columns / Workflow                          | Automation Features                                      | Agile Suitability |
|------------------|---------------------------------------------|----------------------------------------------------------|-------------------|
| Basic Kanban     | To Do, In Progress, Done                    | No automation                                            | Good for simple task tracking but lacks Agile automation |
| Automated Kanban | To Do, In Progress, Done                    | Automatically moves issues based on status               | Highly suitable for Agile (supports continuous workflow) |
| Bug Triage       | New, Needs Review, Confirmed, Fixed         | Helps prioritize and categorize bugs                     | Suitable for maintenance phase, not full Agile workflow |
| Team Planning    | Backlog, Ready, In Progress, Done           | Supports planning and prioritization                     | Good for sprint planning but limited automation          |

## ✅ Selected Template: Automated Kanban

The **Automated Kanban** template was selected for the Student Assignment Tracker system because it aligns well with Agile development practices.

Firstly, it provides built-in automation that automatically moves issues across columns (e.g., from "To Do" to "In Progress" and "Done"), reducing manual effort and improving workflow efficiency.

Secondly, it supports continuous delivery and iterative development, which are key Agile principles. This is important for the project since features such as assignment creation, reminders, and tracking will be developed incrementally.

Additionally, the automated workflow ensures better task visibility and progress tracking, making it easier to manage user stories created in Assignment 6.

Therefore, Automated Kanban is the most suitable template as it enhances productivity, supports Agile methodology, and integrates seamlessly with GitHub Issues.
