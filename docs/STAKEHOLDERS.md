# Stakeholder Analysis – Student Assignment Tracker

## Overview

This document identifies the key stakeholders of the Student Assignment Tracker system, along with their roles, concerns, pain points, and success metrics.

---

## Stakeholder Analysis Table

| Stakeholder               | Role                                               | Key Concerns                                                         | Pain Points                                                 | Success Metrics                                           |
| ------------------------- | -------------------------------------------------- | -------------------------------------------------------------------- | ----------------------------------------------------------- | --------------------------------------------------------- |
| Student                   | Uses the system to track assignments and deadlines | Easy access to assignments, timely notifications, usability          | Missing deadlines, poor organization, scattered information | Reduced missed deadlines, improved task completion rate   |
| Lecturer                  | Creates and manages assignments                    | Efficient assignment distribution, clear communication with students | Manual distribution of assignments, lack of tracking        | Faster assignment publishing, improved student engagement |
| System Administrator      | Manages system configuration and access            | System reliability, user management, security                        | Managing user roles manually, system downtime               | High system uptime (≥99%), smooth user management         |
| University Administration | Oversees academic processes                        | Monitoring student performance and compliance                        | Lack of centralized data and reporting                      | Improved reporting accuracy and decision-making           |
| IT Support Staff          | Maintains and supports the system                  | System stability, maintainability, issue resolution                  | Frequent system issues, lack of documentation               | Reduced system downtime, faster issue resolution time     |
| External Systems (LMS)    | Integration with other academic systems            | Data consistency, seamless integration                               | Data duplication, integration complexity                    | Successful system integration and data synchronization    |
