# 🪞 Reflection

## ✅ Challenges Faced

One of the main challenges in this assignment was selecting the most suitable GitHub project template. Initially, multiple templates such as Basic Kanban, Automated Kanban, and Team Planning appeared similar, but understanding their differences in terms of automation and workflow required deeper analysis.

Another challenge was customizing the Kanban board to accurately reflect a real Agile workflow. Deciding which columns to add, such as "Testing" and "Blocked", required thinking beyond the default setup and considering how tasks move in a real software development environment.

Additionally, linking issues, assigning tasks, and organizing them across different stages required careful planning to ensure the board was not only functional but also realistic and easy to understand.

---

## ✅ Comparison with Other Tools

Compared to other project management tools like Trello and Jira, GitHub Projects offers a more integrated experience for developers. Since it is directly connected to the repository, issues, and commits, it allows seamless tracking of tasks without switching platforms.

Trello is more user-friendly and visually simple, making it suitable for beginners, but it lacks the deep integration with code that GitHub provides. Jira, on the other hand, is more powerful and feature-rich, especially for large teams and complex projects, but it can be more complex to set up and use.

GitHub Projects provides a balanced solution by combining simplicity with essential Agile features, making it ideal for small to medium-sized projects like the Student Assignment Tracker.

---

## ✅ Lessons Learned

This assignment helped me understand how Kanban boards are used in real-world Agile development to manage tasks and visualize progress. I learned the importance of structuring workflows properly, limiting work-in-progress, and clearly defining task stages.

I also gained practical experience using GitHub Projects to manage development tasks, link issues, and track progress, which are essential skills for collaborative software development.

Overall, this assignment improved my understanding of Agile project management and how tools like GitHub can be used effectively to support it.
