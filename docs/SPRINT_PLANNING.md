## Sprint Planning

### Sprint Goal

The goal of this sprint is to deliver the core functionality of the Student Assignment Tracker system by implementing user authentication and essential assignment management features. This will enable lecturers to create assignments and students to securely access and view them, forming the foundation of the system’s minimum viable product (MVP).

---

### Sprint Duration

Start Date: 05/04/2026
End Date: 19/04/2026

Duration: 2 Weeks

---

### Sprint Backlog

The following user stories have been selected for this sprint:

* US-001: User Registration
* US-002: User Login
* US-003: Create Assignment
* US-004: View Assignments
* US-007: Track Assignment Deadlines

---

### Prioritisation

User stories have been prioritised based on their importance to the system’s core functionality:

**High Priority**

* US-001: User Registration
* US-002: User Login

**Medium Priority**

* US-003: Create Assignment
* US-004: View Assignments

**Low Priority**

* US-007: Track Assignment Deadlines

High-priority items are essential for system access and security, while medium-priority items enable core functionality. Low-priority items enhance usability but are not critical for the MVP.

---

### Effort Estimation (Story Points)

Effort estimation has been conducted using story points based on complexity, effort, and time required:

* US-001: User Registration – 5 Story Points
* US-002: User Login – 5 Story Points
* US-003: Create Assignment – 8 Story Points
* US-004: View Assignments – 5 Story Points
* US-007: Track Assignment Deadlines – 3 Story Points

**Total Effort: 26 Story Points**

This workload is considered achievable within the sprint duration based on the scope and complexity of the tasks.

---

### Task Breakdown

Each user story has been decomposed into smaller, manageable tasks:

#### US-001: User Registration

* Design registration form interface
* Implement backend validation logic
* Store user data securely in the database

#### US-002: User Login

* Design login interface
* Implement authentication functionality
* Handle login errors and validation

#### US-003: Create Assignment

* Design assignment creation form
* Implement backend logic for assignment creation
* Store assignment data in the database

#### US-004: View Assignments

* Develop assignment listing interface
* Retrieve assignment data from the database
* Display assignment details to users

#### US-007: Track Assignment Deadlines

* Implement deadline tracking functionality
* Highlight upcoming and overdue assignments

---

### Sprint Planning Justification

The selected user stories were chosen to ensure delivery of a functional MVP. User authentication (registration and login) is critical for secure access, while assignment creation and viewing provide the primary system functionality.

The inclusion of deadline tracking enhances usability and supports the system’s goal of helping students manage their academic workload effectively.

The total estimated effort of 26 story points is appropriate for a 2-week sprint and reflects a balanced workload that is both realistic and achievable.

### Definition of Done

A user story is considered complete when:

* The feature has been fully implemented and is functional
* All acceptance criteria have been satisfied
* No critical errors or defects are present
* Code has been committed and pushed to the GitHub repository
* The corresponding issue has been updated and marked as complete on the project board

---

### Sprint Reflection

This sprint is focused on delivering the foundational components of the Student Assignment Tracker system. By completing the selected user stories, the system will achieve a working MVP capable of supporting both students and lecturers.

Future sprints will focus on extending functionality, including features such as assignment submissions, notifications, and enhanced user experience.
