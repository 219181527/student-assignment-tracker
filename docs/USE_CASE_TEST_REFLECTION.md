# Reflection – Use Case Modeling and Test Case Development

## Overview

This reflection discusses the challenges and insights gained while translating system requirements into use case diagrams, detailed use case specifications, and test cases for the Student Assignment Tracker system.

---

## Challenges in Translating Requirements to Use Cases

One of the main challenges was converting functional requirements into meaningful use cases. While the functional requirements defined what the system should do, it required careful thought to model how different users would interact with the system.

For example, distinguishing between actions performed by students and lecturers required a clear understanding of user roles. Some requirements, such as notifications and deadline tracking, also required interpretation to determine how they should be represented as user interactions within the system.

---

## Challenges in Developing Use Case Specifications

Writing detailed use case specifications required a structured approach. It was sometimes difficult to clearly define preconditions and postconditions without overlapping them with other use cases.

Additionally, identifying alternative flows was challenging because it required thinking about possible system failures or user errors, such as invalid inputs or missing data. This process highlighted the importance of considering edge cases during system design.

---

## Challenges in Designing Test Cases

Developing test cases required mapping each functional requirement to a test scenario. Ensuring traceability between requirements and test cases was an important but sometimes complex task.

Another challenge was defining clear and measurable expected results. For example, performance-related requirements required specific values, such as response time limits, to make them testable.

Designing non-functional test cases, such as performance and security tests, also required a broader understanding of system behavior beyond basic functionality.

---

## Lessons Learned

This assignment provided valuable insights into the importance of structured system design. It demonstrated how requirements engineering, use case modeling, and testing are interconnected.

I learned that:

* Clear requirements lead to better system design
* Use case diagrams help visualize user interactions effectively
* Test cases ensure that the system meets both functional and non-functional requirements

---

## Conclusion

Overall, this assignment improved my understanding of how to translate abstract requirements into practical system models and validation processes. It emphasized the importance of attention to detail, structured thinking, and aligning system design with stakeholder needs.
