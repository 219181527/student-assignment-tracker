# Student Assignment Tracker – System Specification

## 1. Introduction

### 1.1 Project Title

Student Assignment Tracker

### 1.2 Domain

Education Technology (EdTech)

The system operates within the **education domain** and focuses on helping students and lecturers manage academic assignments and deadlines. Educational institutions often require students to track multiple assignments across different courses. Without an organized system, students may miss deadlines or struggle to manage their workload effectively.

This system provides a centralized platform that allows students to monitor their assignments and deadlines while enabling lecturers to publish and manage coursework.

---

## 2. Problem Statement

Students often struggle to manage multiple assignments across different courses, especially when deadlines overlap or are frequently updated. Traditional methods such as manual tracking, spreadsheets, or email notifications are often inefficient and lead to missed submissions or poor time management.

Lecturers also require a structured way to publish assignments and ensure students are aware of deadlines.

The **Student Assignment Tracker** aims to solve this problem by providing a digital system where assignments can be created, tracked, and managed in a centralized environment.

---

## 3. System Overview

The system will allow two main types of users:

### Students

Students will be able to:

* View assignments posted by lecturers
* Track assignment deadlines
* Mark assignments as completed or submitted
* Organize assignments across courses

### Lecturers

Lecturers will be able to:

* Create assignments
* Set due dates
* Manage coursework for their modules
* Update assignment details

---

## 4. Individual Scope

This project is designed to be **feasible for an individual student to design and document within the semester**. The focus of the project is on:

* Defining a clear **system specification**
* Designing **software architecture using the C4 model**
* Demonstrating how system components interact

The system will be designed as a **web-based application** consisting of the following high-level components:

* User interface (web application)
* Backend application server
* Database for storing users and assignments
* Authentication component for managing user access

The implementation itself is not the primary focus; rather, the emphasis is on **clear documentation, system design, and architectural modeling**.
